## 1. 环境要求

| 依赖 | 通用要求 | 用途 |
|------|----------|------|
| Node.js | `>= 18` | 前端构建与 Vite 开发服务 |
| Rust | stable toolchain | Tauri/Rust 后端编译 |
| MSVC Build Tools | 已安装 C++ build tools | Windows 下 Tauri 原生依赖编译 |
| Windows SDK | 已安装并可被 MSVC 正确发现 | 系统头文件与链接库 |

### 1.1 为什么需要 MSVC 而不是 GNU?

Tauri 在 Windows 上依赖 WebView2，其链接规模不适合 GNU 方案中的常见 Windows PE 限制场景。
如果使用 GNU 工具链，可能出现 `export ordinal too large` 等链接错误，因此建议统一使用 `stable-x86_64-pc-windows-msvc` 工具链。

### 1.2 为什么建议优先使用 PowerShell / Developer PowerShell?

在 Windows 上，Visual Studio 的 Developer PowerShell / Developer Command Prompt 会自动注入 MSVC 所需环境变量。

如果使用普通 PowerShell 或 Git Bash，则可能需要手动补齐：

- `PATH`
- `INCLUDE`
- `LIB`

因此：

- 日常开发优先使用 **PowerShell** 或 **Developer PowerShell**
- 只有在 Git Bash 中编译时，才建议手动设置相关环境变量

## 2. 编译前环境设置

### 2.1 推荐方式：直接使用已配置好 MSVC 的终端

优先在以下终端中执行命令：

- Visual Studio Developer PowerShell
- Visual Studio Developer Command Prompt
- 已手动注入 MSVC 环境变量的普通 PowerShell

### 2.2 Git Bash / 普通终端的通用模板

如果你必须在 Git Bash 中执行，可按实际安装位置填入以下占位符：

```bash
# Rust cargo bin
export PATH="<RUSTUP_HOME>/.cargo/bin:<MSVC_BIN_DIR>:$PATH"

# MSVC include
export INCLUDE="<MSVC_INCLUDE_DIR>;<WINDOWS_SDK_UCRT_INCLUDE_DIR>;<WINDOWS_SDK_UM_INCLUDE_DIR>;<WINDOWS_SDK_SHARED_INCLUDE_DIR>"

# MSVC lib
export LIB="<MSVC_LIB_X64_DIR>;<WINDOWS_SDK_UCRT_LIB_X64_DIR>;<WINDOWS_SDK_UM_LIB_X64_DIR>"
```

其中：

- `<RUSTUP_HOME>`：Rust 安装根目录
- `<MSVC_BIN_DIR>`：MSVC `Hostx64/x64` 编译器目录
- `<MSVC_INCLUDE_DIR>`：MSVC `include` 目录
- `<MSVC_LIB_X64_DIR>`：MSVC `lib/x64` 目录
- `<WINDOWS_SDK_*>`：Windows SDK 对应目录

> 提示：如果你的机器已经正确安装 Visual Studio Build Tools，通常优先使用 Developer PowerShell，更稳定，也更不容易因为路径拼错导致编译失败。

## 3. 编译命令

以下命令中的 `<PROJECT_ROOT>` 表示仓库根目录。

### 3.1 开发模式（热更新）

```powershell
Set-Location <PROJECT_ROOT>
npx tauri dev
```

这会同时启动：

- Vite 前端开发服务器，默认地址为 `http://localhost:1420`
- Tauri 桌面壳层
- Rust 后端增量编译

### 3.2 仅运行前端开发服务

```powershell
Set-Location <PROJECT_ROOT>
npm.cmd run dev
```

### 3.3 仅构建前端

```powershell
Set-Location <PROJECT_ROOT>
npm.cmd run build
```

### 3.4 仅检查 Rust / Tauri

```powershell
Set-Location <PROJECT_ROOT>\src-tauri
cargo check
```

### 3.5 运行测试

```powershell
Set-Location <PROJECT_ROOT>
npm.cmd test
cargo test --manifest-path src-tauri/Cargo.toml
```

### 3.6 生产构建

```powershell
Set-Location <PROJECT_ROOT>
npx tauri build
```

构建产物默认位于：

- `src-tauri/target/release/`

## 4. 在线测试与调试

### 4.1 浏览器调试

只调试前端界面时：

```powershell
Set-Location <PROJECT_ROOT>
npm.cmd run dev
```

访问：

- `http://localhost:1420`

### 4.2 Tauri 桌面端调试

需要完整桌面壳层时：

```powershell
Set-Location <PROJECT_ROOT>
npx tauri dev
```

### 4.3 端口与进程检查

在重启服务前，应先检查端口或进程占用情况，不要直接结束整个 shell，也不要直接结束全部 `node.exe`。

检查 `1420` 端口：

```powershell
netstat -ano | Select-String ':1420'
```

按 PID 结束占用进程：

```powershell
taskkill /F /PID <PID>
```

检查 GetChat 相关进程：

```powershell
tasklist | findstr /i "getchat"
```

## 5. 常见问题

### Q: 报错 `cargo: command not found`

**原因**：Rust 未安装，或 cargo bin 未进入 `PATH`。

**解决**：

```powershell
cargo --version
rustc --version
```

如果命令不存在，先补齐 Rust 环境后再执行编译。

### Q: 报错 `link.exe: command not found` 或 `error: linker 'link.exe' not found`

**原因**：MSVC 编译环境未正确注入。

**解决**：

- 优先换用 Developer PowerShell
- 或手动设置 `PATH` / `INCLUDE` / `LIB`

### Q: 报错 `export ordinal too large`

**原因**：通常说明当前使用了 GNU 工具链而不是 MSVC。

**解决**：

- 确认 Rust toolchain 为 `stable-x86_64-pc-windows-msvc`
- 确认当前终端实际调用的是 MSVC 链接器

### Q: 报错 `sqlite3.h: No such file or directory`

**原因**：SQLite 未走 bundled 编译链路，或本机工具链异常。

**解决**：确认 [`src-tauri/Cargo.toml`](../src-tauri/Cargo.toml) 中包含：

```toml
libsqlite3-sys = { version = "0.30", features = ["bundled"] }
```

### Q: 数据库相关错误 `DB_ERROR`

**可能原因**：

1. 运行时数据库损坏
2. 迁移脚本异常
3. 本地旧数据与当前 schema 不兼容

**排查重点**：

- `src-tauri/src/db/migrations/`
- `%APPDATA%\com.getchat.app\`

### Q: 前端端口被占用

先查 PID：

```powershell
netstat -ano | Select-String ':1420'
```

再按 PID 结束：

```powershell
taskkill /F /PID <PID>
```

## 7. Logo / Icon 重建

修改品牌源文件后，可重新生成桌面端图标并同步浏览器 favicon：

```powershell
Set-Location <PROJECT_ROOT>
npx tauri icon src/assets/brand/getchat-logo.svg -o src-tauri/icons
Copy-Item -LiteralPath src-tauri/icons/icon.ico -Destination public/favicon.ico -Force
```

说明：

- 浏览器 SVG favicon 使用 `public/favicon.svg`
- 桌面端打包图标使用 `src-tauri/icons/*`

## 8. UTF-8 编码约束

Windows 环境下，所有新增或修改的文本文件都应显式使用 UTF-8，避免中文乱码。

PowerShell 读取：

```powershell
Get-Content -Raw -Encoding UTF8 <path>
```

PowerShell 写入：

```powershell
Set-Content -Encoding UTF8 <path> <content>
```

HTML 入口应显式声明：

```html
<meta charset="UTF-8" />
```
